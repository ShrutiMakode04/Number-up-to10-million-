1. EXECUTIVE SUMMARY

This document defines product requirements for the "Equal Groups — Introduction
to Multiplication and Division" interactive lesson module, delivered as Lesson 4.1
within Intellia SG's Grade 1 Math program. The module targets Singapore Primary 1
students aged 6–7 and introduces the foundational concept of equal groups as the
concrete and pictorial gateway to multiplication and division thinking, fully aligned
with the Singapore MOE Mathematics Framework and the Primary 1 Targeting
Mathematics syllabus.

The product is a standalone web page to be hosted at:
  https://intelliasg.com/courses/grade-1-math/lessons/equal-groups/

It is built using React (Vite + JSX, JavaScript/CSS) and designed to strictly mirror
the visual and UX structure established at https://numberbound.vercel.app/ and the
repository https://github.com/dsamyak/numberbound.

Audio narration uses ElevenLabs exclusively (Voice: Alice, Voice ID: Xb7hH8MSUJpSbSDYk0k2,
Model: eleven_multilingual_v2) with pre-generated static .mp3 files for all phase
narrations and dynamic generation for practice questions — matching the pipeline
documented in the Word Problems Using Addition Audio & Narration Pipeline.

The module follows Intellia's 5-phase learner journey:

  Phase 1 — INTRO      → Welcome screen + 5-phase progress map
  Phase 2 — WONDER     → Curiosity hook
  Phase 3 — STORY      → Narrative-based concept introduction
  Phase 4 — SIMULATE   → Sandbox-style interactive simulation (3 stations)
  Phase 5 — PLAY       → IntelliPlay™ gamified practice (100 randomised questions)
  Phase 6 — REFLECT    → Journal / LearnFlow AI prompt + completion badge

  - Singapore MOE Primary Mathematics Syllabus (Primary 1–6), 2013 (Revised)
    → Strand: Number and Algebra

  - My Pals Are Here! Maths Primary 1B (Marshall Cavendish)
    → Unit 14: Multiplication — Making Equal Groups

MOE Learning Objectives Covered (Lesson 4.1):

  LO1  Recognise and describe equal groups (same number in each group)
  LO2  Count the total number of objects in equal groups by repeated counting
  LO3  Represent equal groups using pictures and diagrams
  LO4  Connect equal groups to repeated addition (e.g. 3 groups of 4 = 4+4+4)
  LO5  Describe sharing a set of objects equally into groups (fair sharing / division concept)
  LO6  Use language: "groups of", "equal groups", "each group has", "how many altogether"
  LO7  Solve simple picture-based word problems involving equal groups

Singapore Syllabus CPA Progression for This Lesson:

  Concrete    → Physical counters/objects arranged into equal groups (simulated digitally)
  Pictorial   → Diagrams of grouped objects, dot arrays, equal-group cards
  Abstract    → "3 groups of 4 = ___"; connecting to repeated addition 4+4+4=12

Number Ranges:
  Easy:    Groups of 2, 3, 4, 5 (total ≤ 20)
  Medium:  Groups of 2–6 (total ≤ 30)
  Hard:    Groups of 2–9 (total ≤ 40); includes reverse (find groups or group size)

Vocabulary Focus (age-appropriate):
  "equal groups", "same number", "groups of", "each", "altogether",
  "share equally", "how many in each group", "how many groups"

6. PHASE 3 — SIMULATION DESIGN (Detailed)

6.1 Station A — The Grouping Table (Concrete)

Visual:
  - A cheerful table/mat area divided into 2–5 equal circles (group zones)
  - A pool of draggable emoji objects on the left (stars, fish, cookies, etc.)
  - "Put ___ in each group!" instruction narrated by Alice (ElevenLabs)

Interaction:
  - Student drags objects one at a time into each circle
  - OR taps an object + taps destination circle (accessibility mode)
  - Each circle shows a counter "0 / 3" updating as objects are placed
  - A number group diagram (bubble visual) updates in real-time below

Feedback:
  - All groups equal → mascot dances, "Amazing! Every group has the same!" 🎉
  - Unequal groups on submit → gentle shake + "One group has more! Let's fix it."

Variants per round (randomised):
  - Round 1: 2 groups of 3 (6 objects)
  - Round 2: 3 groups of 4 (12 objects)
  - Round 3: 4 groups of 2 (8 objects)
  - Round 4: 5 groups of 3 (15 objects)

6.2 Station B — Spot the Equal Groups (Pictorial)

Visual:
  - 4 arrangement cards displayed on screen (2×2 grid)
  - Each card shows groups of emoji objects arranged in bubbles
  - Some cards have EQUAL groups; some are deliberately unequal

Interaction:
  - Student taps the card(s) where all groups have equal counts
  - Multi-select: correct cards glow green, wrong cards glow red on submit
  - Diagrams are shown without numbers (pure visual discrimination)

Teaching goal:
  - Equal vs. unequal — building the core conceptual distinction
  - Encourages careful visual counting without explicit arithmetic

Distractor design:
  - Unequal cards differ by ±1 in one group (plausible, not obvious)
  - One card always has a larger but unequal arrangement (catches skimmers)

3 rounds with increasing visual complexity:
  - Round 1: 2-group arrangements (easy visual split)
  - Round 2: 3-group arrangements (medium)
  - Round 3: 4-group arrangements (hard — tests careful counting)

6.3 Station C — Fill the Sentence (Abstract)

Visual:
  ___ groups of ___ = ___
  (one blank highlighted for input; the other two values visible)

Interaction:
  - Number pad (large, tap-friendly, 0–9 displayed)
  - Group diagram shown above as visual scaffold
  - "Show diagram" hint button always visible
  - On submit: correct → bounce animation; incorrect → shake + hint

Variants (which blank is missing, rotated per round):
  - Find total:    3 groups of 4 = ___
  - Find group size: 2 groups of ___ = 10
  - Find number of groups: ___ groups of 5 = 15

ElevenLabs narrates each sentence aloud when displayed.

8. GAMIFICATION DESIGN

8.1 Reward System

  Stars (⭐):    Earned per 10-question world (1–3 stars based on score)
  XP Points:    10 XP correct first try | 7 XP second try | 5 XP with hint used
  Streak 🔥:    Fire counter for consecutive correct answers
  Streak Bonus: +5 XP per correct answer when streak ≥ 5

8.2 Badges (Unlockable)

  🏅 "Group Explorer"    — Complete Wonder + Story phases
  🥈 "Group Builder"     — Complete all 3 Simulation stations
  🥇 "Equal Champion"   — Score ≥80% on Play phase (≥80 questions correct)
  💎 "Perfect Groups"   — Score 10/10 in any world
  🔥 "Streak Star"      — Achieve a streak of 10 consecutive correct answers
  🌟 "Full Journey"     — Complete all 5 phases (lesson complete badge)
  🎯 "Sharp Eye"        — Get 5 correct in Station B without any wrong pick
  🤝 "Fair Sharer"      — Answer 5 fair-sharing word problems correctly (Q7)

8.3 Feedback Mechanics

  ✅ Correct:
     - Bounce animation on answer card + mascot happy mood
     - ElevenLabs celebration audio: "Amazing! You made equal groups! 🎉"
     - XP floats up from answer card (+10 / +7 / +5)
     - Streak fire counter increments

  ❌ Incorrect (Attempt 1):
     - Gentle shake animation on answer card
     - ElevenLabs gentle voice: "Not quite! Let's look at the groups again 🔢"
     - Hint 1 activates: Group diagram highlighted with circle counts shown

  ❌ Incorrect (Attempt 2):
     - Stronger shake + Hint 2: Counter animation shows grouping visually
     - ElevenLabs: "Watch me group them! Can you count with me?"

  ❌ Incorrect (Attempt 3):
     - Answer revealed with animated explanation (mascot explains)
     - ElevenLabs: full explanation read aloud
     - No score penalty — encouragement only

  No negative scoring. Encouragement-first approach always.

9.3 Pre-generated Audio Files (Offline — scripts/generate_audio.js)

  All phase narration lines (Wonder, Story panels, Simulate instructions,
  Reflect prompt, badge unlock messages, world completion) are pre-generated
  offline and stored as static .mp3 in public/assets/audio/.

  audioMap.js is auto-generated and maps exact text strings → file paths.
  The frontend checks audioMap first; dynamic generation only for play-phase
  questions not in the map (using elevenLabsCache in memory).

9.4 Dynamic Generation

  Practice questions (Phase 4) are generated dynamically if not pre-cached.
  Requires VITE_ELEVENLABS_API_KEY in .env.local.
  If key is absent, narration silently skipped (no browser TTS fallback).
  Internal memory cache (elevenLabsCache) prevents re-fetching same text.

9.5 Segment Synchronisation

  The audio engine parses narration as an array of segments (one per sentence).
  While segment i plays, segment i+1 is eagerly preloaded via getAudioUrl.
  This guarantees seamless, gap-free narration across multi-sentence scripts.
  Uses HTML5 Audio API (new Audio()) for playback.

11. CONTENT REQUIREMENTS

11.1 Simulation Visuals

  Group diagrams:      SVG-rendered large circles with emoji dot objects inside
  Object pool:         Colourful emoji items (🍓 🌟 🍪 🐟 🎈 🍎) rotated per session
  Station B cards:     4 arrangement cards, distinct layouts, clean whitespace
  Abstract sentences:  Large bold typography, one highlighted blank per round

11.2 Question Bank Coverage

  All 10 question types × 10 questions = 100 unique question objects in questionBank.js
  Questions randomised per session using Fisher-Yates shuffle
  No two sessions present same question order
  MCQ distractors always plausible (within ±3 or logical wrong group count)

11.3 Word Problem Format (MOE style)

  Multiplication sense:
    "[Name] puts [group-size] [objects] on each [container].
     [He/She] has [num-groups] [containers]. How many [objects] altogether?"

  Fair-sharing / Division sense:
    "[Name] has [total] [objects]. [He/She] shares them equally into
     [num-groups] [containers]. How many [objects] does each [container] get?"

  Let's Think Along style (extended):
    "[Name] has ___ [objects]. [He/She] makes ___ equal groups.
     How many are in each group? Draw it and check!"

11.4 Audio Script Parity (1:1 Strict Parity Rule)

  Every on-screen text string that is narrated must match the narration.js text
  exactly — same words, same punctuation. This prevents confusion for young learners
  who are simultaneously listening and reading. Any UI text change requires updating
  both the generate_audio.js phrases array and the narration.js file.
